# Scope and Boundaries

This document defines exactly what is and is not included in each package. It exists to prevent scope creep, protect working hours, and ensure every client engagement is sustainable long-term.

Read this before signing any client. Reference it when a client requests something outside the defined scope.

---

## Definition of a "Working Day"

A working day is not a calendar day of availability.

A working day is a **pre-agreed structured work block** of approximately 4 hours with:
- A defined list of outputs agreed in advance
- A fixed time window communicated at the start of the month
- A clear scope of what will be completed

### What counts as a structured block output

- Job description created or updated (maximum 3 per block)
- New hire org setup (role created, assignment recorded, onboarding checklist initialised)
- Role change processed (reporting line, department, title change)
- Compliance document status updated
- Org structure review and gap report
- Policy document drafted (maximum 1 per block)

### What does not count as a structured block

- Open-ended availability ("just be around in case")
- Attending internal meetings beyond a single briefing
- Responding to ad-hoc Slack messages during the block
- Tasks not agreed at the start of the month

## What Is Never Included (Any Package)

- 24/7 availability or out-of-hours response
- Employment law advice or legal counsel (always refer to a qualified employment solicitor)
- Payroll processing or payroll advice
- Recruitment sourcing (finding and screening candidates)
- Performance improvement plan management
- Disciplinary or grievance process management
- Immigration or visa advice
- Benefits administration
- Unlimited Slack or WhatsApp support outside async windows

## Communication Rules

### Default mode: async

All communication is asynchronous by default. Responses within 48 hours for Package 1. Responses within 24 hours for Packages 2 and 3 during defined working windows.

### Real-time availability

Real-time availability (calls, messages requiring immediate response) is only available during pre-agreed structured blocks (Packages 2 and 3).

### Monthly planning

At the start of each month, Package 2 and Package 3 clients receive:
- Proposed block dates for the month
- Draft output list for each block
- Confirmation of async channel and response window

This is agreed before the month begins. Changes to the output list require 5 working days notice.

## Preventing Scope Creep

These patterns occur in almost every client engagement. Address them early.

### Pattern 1 — The expanding request
> "While you're updating that role, could you also just quickly..."

Response: "Happy to add that — it will come from next month's block allocation or I can quote it as an additional item."

### Pattern 2 — The after-hours message
> [WhatsApp message at 8pm] "Quick question about X..."

Response: "I'll pick this up during my next working window on [date]. If it's urgent and outside our agreed schedule, let me know and we can discuss adding an ad-hoc session."

### Pattern 3 — The undefined request
> "Can you just sort out our HR situation generally?"

Response: "Let me send you a proposed output list for this month so we're aligned on exactly what we'll deliver."

### Pattern 4 — The meeting invitation
> [Calendar invite for standing weekly meeting]

Response: "I don't include standing meetings in the retainer scope. I can join a single briefing at the start of each month to agree outputs. Additional meetings would come from block allocation."

## Contract Minimum

All clients sign a 3-month minimum commitment before work begins. This is non-negotiable. It protects the setup investment and ensures the system has time to embed before the client evaluates value.
