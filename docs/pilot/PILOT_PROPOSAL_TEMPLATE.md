## Section 2 — What this system does

TeamFrame creates operational legibility by:
- making org structure visible
- enforcing ownership clarity
- converting strategy into tracked actions
- attaching policies directly to organizational structure

Expected outcome:
- leadership can answer structure/ownership/urgency/policy questions quickly and consistently.

## Section 3 — Pilot scope (explicit boundaries)

Pilot includes:
- fixed onboarding process
- deterministic setup and walkthrough
- clarity validation using defined scorecard

Pilot excludes:
- customization outside core flow
- custom feature work
- integrations

Window:
- fixed 1-2 week activation cycle

## Section 4 — Success definition

Pilot is considered successful when:
- COO clarity is achieved
- full org is mapped in system
- actions are actively tracked with explicit ownership
- ownership is visible across structure

Primary success test:
- stakeholder answers the 4 clarity questions in under 5 minutes inside TeamFrame.

## Section 5 — What is NOT included

- performance management systems
- workflow automation design
- custom integrations

Boundary statement:
> This pilot validates operational clarity. It is not a full systems transformation engagement.
